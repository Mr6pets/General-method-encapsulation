## git 代码仓库 提交代码 日志信息，格式逐步格式规范，具体GIT提交记录信息描述格式如下：

~~~js



1、如果有对应需求，记录信息描述格式
 #STORY id=需求编号 +空格+ 需求标题+其他备注说明

2、如果有对应执行，提交格式 
 #EXECUTION id=执行编号 +空格+ 执行标题 +其他备注说明
 
3、如果有对应执行任务，记录信息描述格式
 #TASK id=任务编号 +空格+ 任务标题 +其他备注说明

4、如果有对应测试BUG，记录信息描述格式
 #BUG id=BUG编号 +空格+BUG标题 +其他备注说明

备注：
1、代码提交可写多个需求或BUG，每个以#打头，尽量细分后提交，标题可以是开发页面或功能名称及描述，其他备注说明 可以不写；
2、开发的代码如果项目管理没有对应记录，可以对应新建好记录后再提交相关代码；
3、提交master后的代码，可以按计划通知更新正式服或打包发布版本并通知相关人员测试确认；

~~~
